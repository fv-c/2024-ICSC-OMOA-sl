Foreword

About the Reverberators project (1)

The Reverberators project was born in the educational context of the “N. Piccinni” Conservatory of Bari (IT).

The starting point was identified in Manfred Schroeder's literary works on the creation of artificial reverberations with qualities similar to natural ones. [1, 2]

About the Reverberators project (2)

Those works reports reverberator's block diagrams and impulse response (IR).

The project was therefore a path of reconstruction and conscious implementation [3] of historical reverberators, with a strong view to their creative expansion.

Why Faust (Functional Audio Stream) initially?

  • text-based programming language
  • highest level functions are implemented using the same language
    (up to the basic “primitive” components)
  • it has tools to analyze sample-by-sample and visualize block diagrams

Why analysis tools?

Faust's analytical ability had allowed us to adopt a series of compensation strategies for the delay samples that did not emerge from the block diagram reported in the literature.
This was possible by analyzing the IR of our processors, comparing it with the one present in the papers, making the analytical tool essential for an in-depth understanding of the reconstruction process.

Csound porting

Educational motivations

Csound has many affinities with Faust: it is a text-based system, which allows for careful analysis of the flow of operations on samples; furthermore the definition of functions (user defined opcodes) can be done using the csound language itself.
Another detail is that the Csound language has long been an integral part of our school's teaching.
